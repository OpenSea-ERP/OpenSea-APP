# Auditoria de Responsividade - Modulo Estoque

**Data:** 2026-03-10
**Modulo:** Stock (Estoque)
**Escopo:** `src/app/(dashboard)/(modules)/stock/` e componentes compartilhados
**Auditor:** Claude Opus 4.6

---

## Resumo Executivo

O modulo de estoque apresenta uma abordagem parcialmente responsiva. Os componentes compartilhados (PageHeroBanner, PageDashboardSections, EntityGrid) possuem breakpoints adequados, porem diversas paginas internas utilizam grids fixos e layouts que nao se adaptam a telas menores que 768px. O layout principal (`pt-28 px-6`) nao reduz o padding horizontal em mobile, consumindo 48px de espaco valioso em telas de 320px.

**Nota final: 5.4 / 10.0**

## Analise Detalhada por Criterio

### 1. Abordagem Mobile-First

**Resultado: WARN**

O projeto utiliza TailwindCSS 4 com breakpoints padrao (sm: 640px, md: 768px, lg: 1024px, xl: 1280px). Os componentes compartilhados seguem mobile-first (ex: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`), porem diversas paginas de entidade utilizam grids fixos sem prefixo responsivo:

- `src/app/(dashboard)/(modules)/stock/(entities)/products/[id]/edit/page.tsx` linha 440: `grid grid-cols-3 gap-4` (sem breakpoint)
- Mesma pagina, linha 504: `grid grid-cols-2 gap-4` (sem breakpoint)
- Mesma pagina, linha 552: `grid grid-cols-3 gap-4` (atributos personalizados, sem breakpoint)

Essas classes aplicam layout multi-coluna desde 320px, o que causa compressao severa dos campos de formulario.

### 2. Overflow Horizontal em Mobile (320px-480px)

**Resultado: FAIL (Critico)**

Problemas identificados:

1. **Layout principal** (`src/app/(dashboard)/layout.tsx`, linha 108): `className="pt-28 px-6 pb-12"` aplica `padding: 0 24px` em todas as telas. Em 320px, isso deixa apenas 272px de largura util. Deveria ser `px-3 sm:px-4 md:px-6`.

2. **Stock Overview - Tabela** (`src/app/(dashboard)/(modules)/stock/overview/list/page.tsx`): Tabela com `table-fixed` e colunas de largura fixa (48px + auto + 180px + 180px + 160px + N\*120px). Em mobile, a soma minima e ~588px sem colunas dinamicas, causando overflow horizontal. Embora tenha `overflow-auto` no container, a experiencia de scroll horizontal em tabela nao e adequada para mobile.

3. **Movements Overview**: Mesma situacao da tabela de estoque geral.

4. **Edit Product Page** (`grid-cols-3`): Tres campos em 320px resultam em inputs de ~80px de largura cada, inutilizaveis.

5. **Navbar floating**: `w-[calc(100%-2rem)]` com `px-6` interno. Em 320px, a navbar tem 288px com padding de 48px, sobrando 240px para conteudo. Os botoes da direita (Central, Finance Bell, Print Queue, Notifications, Grid Menu, User Dropdown) podem empilhar ou transbordar.

### 3. Touch Targets (minimo 44x44px)

**Resultado: WARN**

- **Botoes de acao (PageActionBar)**: Usam `size="sm"` (32px de altura), abaixo do minimo de 44px. Arquivo: `src/components/layout/page-action-bar.tsx`, linha 98.
- **Filter buttons**: `FilterDropdown` renderiza botoes que herdam tamanhos padrao do shadcn, tipicamente 36px.
- **Table rows (Stock Overview)**: As linhas da tabela tem `estimateSize: 49px`, adequado.
- **Navbar icons**: `size="icon"` no botao do menu (40x40px padrao shadcn), proximo mas abaixo de 44px.
- **Switch components**: Switches nativos do shadcn tem area clicavel adequada.
- **EntityCard footer buttons**: Botoes com texto tem altura adequada (~40px+), porem o icone sozinho pode ficar pequeno.
- **ScanModeSelector**: `size="sm"` nos botoes, abaixo de 44px.

### 4. Tabelas Responsivas

**Resultado: FAIL**

As duas tabelas principais do modulo (Stock Overview e Movements) utilizam `<table>` HTML nativa com virtualizacao (`@tanstack/react-virtual`). Nao ha alternativa mobile (card layout) nem scroll indicators:

- **Stock Overview** (`overview/list/page.tsx`): Tabela com colunas fixas de largura definida via `<colgroup>`. O container tem `overflow-auto`, porem nao ha indicacao visual de scroll horizontal para o usuario.
- **Movements** (`overview/movements/page.tsx`): Mesmo padrao de tabela nao-responsiva.
- **Nao ha fallback card-based para mobile** em nenhuma das tabelas.

As paginas de grid (Products, Templates, Manufacturers, etc.) utilizam `EntityGrid` que adapta corretamente para `grid-cols-1` em mobile.

### 5. Navegacao em Mobile

**Resultado: WARN**

- **Navbar**: O componente `Navbar` renderiza todos os botoes em linha (`flex items-center gap-2`). Em telas estreitas, os 6+ botoes (Central, Finance, Print, Notifications, Menu, User) podem nao caber. Nao ha hamburger menu ou collapse para mobile.
- **ToolsPanel**: Funciona como modal fullscreen (`fixed inset-4`), adequado para mobile. O grid de navegacao usa `grid-cols-2` como base, adequado.
- **Breadcrumb**: Renderizado em linha completa. Em caminhos longos (ex: Estoque > Localizacoes > Armazem > Zona > Estrutura), pode causar overflow.
- **PageActionBar**: Usa `flex justify-between`. Se o breadcrumb for longo e houver muitos botoes, pode haver sobreposicao.

### 6. Formularios em Mobile

**Resultado: FAIL**

Problemas criticos identificados:

1. **Edit Product** (`products/[id]/edit/page.tsx`):
   - Linha 440: `grid grid-cols-3 gap-4` para Nome, Categoria e Fabricante. Em 320px, cada campo tem ~80px. **Deveria ser `grid-cols-1 md:grid-cols-3`**.
   - Linha 504: `grid grid-cols-2 gap-4` para switches Fora de Linha e Produto Ativo. **Deveria ser `grid-cols-1 sm:grid-cols-2`**.
   - Linha 552: `grid grid-cols-3 gap-4` para atributos personalizados. **Deveria ser `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`**.

2. **Edit Product Header** (linha 313): `flex items-center justify-between` com breadcrumb + botoes. Em mobile, os botoes Excluir e Salvar podem empurrar o breadcrumb.

3. **Locations form** (`locations/page.tsx`, linha 266): `grid grid-cols-2 gap-4` para Codigo e Nome. **Deveria ser `grid-cols-1 sm:grid-cols-2`**.

4. **Create Product Form** (dentro de Dialog `max-w-2xl`): Dialogs usam `max-w-[calc(100%-2rem)]` como base, portanto funcionam em mobile. O formulario interno precisa ser verificado separadamente.

### 7. Modais/Dialogs em Mobile

**Resultado: PASS**

Os dialogs utilizam o componente `DialogContent` do shadcn/radix que aplica:

- `w-full max-w-[calc(100%-2rem)]` como base
- `sm:max-w-lg` para dialogs padrao
- Modais customizados como `ProductVariantsItemsModal` usam `sm:max-w-[1100px]` com fallback para full-width em mobile
- `max-h-[90vh] overflow-y-auto` nos dialogs de Create/Edit Product
- `AlertDialogContent` segue o mesmo padrao responsivo

O `ProductVariantsItemsModal` transiciona de `grid-cols-1` para `md:grid-cols-2`, adequado.

### 8. Legibilidade do Texto

**Resultado: PASS**

- O body usa fonte padrao do sistema (definido via TailwindCSS).
- Titulos: `text-3xl` (~30px) no `Header`, `text-3xl md:text-4xl` no `PageHeroBanner`.
- Corpo: `text-sm` (~14px) para descricoes, `text-base` (~16px) para inputs.
- Labels: `text-xs` (~12px) em badges e metadados, aceitavel como complemento.
- Nenhum texto principal abaixo de 14px que exija zoom.

### 9. Imagens/Media

**Resultado: PASS**

- O modulo de estoque nao utiliza imagens pesadas. Os icones sao vetoriais (Lucide/react-icons).
- Thumbnails de templates usam `next/image` com otimizacao automatica.
- Cores de variantes sao renderizadas via `div` com `backgroundColor` inline, sem problemas de escala.
- Os circulos decorativos do layout principal usam `w-[600px] h-[600px]` com `overflow-hidden` no container, sem causar scroll.

### 10. Adaptacao de Grids

**Resultado: WARN**

**Corretos:**

- `EntityGrid`: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4` (padrao)
- `PageDashboardSections`: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`
- `ToolsPanel` navegacao: `grid-cols-2 md:grid-cols-3 lg:grid-cols-5`
- `ToolsPanel` ferramentas: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`
- `Locations` warehouse grid: `grid gap-4 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4`

**Problematicos:**

- Edit Product: `grid-cols-3` fixo (3 instancias)
- Edit Product switches: `grid-cols-2` fixo
- Locations form: `grid-cols-2` fixo
- Edit Product header: `flex justify-between` sem wrap

### 11. Drag-and-Drop com Alternativa Mobile

**Resultado: WARN**

O `EntityGrid` implementa drag selection (lasso/retangulo de selecao) que depende de eventos de mouse (`mousedown`, `mousemove`, `mouseup`). Nao ha implementacao equivalente para touch events:

- Nao ha `touchstart`/`touchmove`/`touchend` listeners.
- A selecao individual por click funciona em touch.
- Shift+click e Ctrl+click nao sao viaveis em touch sem teclado.
- O `ZoneMap` (mapa de localizacao de zona) pode ter drag interactions que nao foram auditadas em detalhe.

A selecao basica (tap individual) funciona, mas a selecao multipla via drag nao tem alternativa touch.

### 12. Paginacao em Mobile

**Resultado: PASS**

- O modulo utiliza paginacao em `Purchase Orders` via componente `Pagination` dedicado.
- Demais paginas usam carregamento completo (sem paginacao server-side) com busca client-side.
- O `SelectionToolbar` (barra flutuante de selecao) usa `fixed bottom-6 left-1/2 -translate-x-1/2` com conteudo em flex, adaptavel.
- A barra flutuante de selecao no Stock Overview usa o mesmo padrao.

## Recomendacoes Prioritarias

### Prioridade Critica (P0)

1. **Corrigir padding do layout principal**
   - Arquivo: `src/app/(dashboard)/layout.tsx`, linha 108
   - De: `className="pt-28 px-6 pb-12"`
   - Para: `className="pt-28 px-3 sm:px-4 md:px-6 pb-12"`

2. **Corrigir grids fixos no Edit Product**
   - Arquivo: `src/app/(dashboard)/(modules)/stock/(entities)/products/[id]/edit/page.tsx`
   - Linha 440: `grid-cols-3` para `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
   - Linha 504: `grid-cols-2` para `grid-cols-1 sm:grid-cols-2`
   - Linha 552: `grid-cols-3` para `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`

3. **Corrigir grid fixo no form de Locations**
   - Arquivo: `src/app/(dashboard)/(modules)/stock/(entities)/locations/page.tsx`, linha 266
   - De: `grid grid-cols-2 gap-4`
   - Para: `grid grid-cols-1 sm:grid-cols-2 gap-4`

### Prioridade Alta (P1)

4. **Tabelas com layout alternativo para mobile**
   - Stock Overview e Movements: Implementar card layout quando `< md` breakpoint, ou pelo menos adicionar indicadores visuais de scroll horizontal e garantir que a primeira coluna seja sticky.

5. **Navbar responsiva**
   - Colapsar botoes secundarios (Finance Bell, Print Queue, Notifications) em um dropdown ou esconde-los atras do menu principal em telas `< sm`.

6. **Touch targets nos botoes de acao**
   - `PageActionBar`: Aumentar `size="sm"` para `size="default"` ou garantir `min-h-[44px] min-w-[44px]`.

### Prioridade Media (P2)

7. **Breadcrumb com truncamento**
   - Implementar truncamento com `...` para caminhos longos em mobile (mostrar apenas primeiro e ultimo item com ellipsis).

8. **Edit Product header responsivo**
   - Linha 313: Adicionar `flex-wrap` e ajustar layout para empilhar breadcrumb e botoes em mobile.

9. **Drag selection com touch**
   - Adicionar modo de selecao multipla via long-press + tap (padrao mobile).
